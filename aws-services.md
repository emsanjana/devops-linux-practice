### AWS Services Commanly used in DevOps

1. AWS EC2
   - Amazon Elastic Compute Cloud
   - Provides resizable compute capacity in the cloud.
   - Enables launching virtual servers (instances) on-demand.
   - Supports scaling up or down automatically.
   - Uses AMI (Amazon Machine Image) to launch instances quickly.
   - Security via Key Pairs and Security Groups.
2. AWS IAM
   - Amazon Identity and Access Management
   - Manages users, groups, and roles in AWS.
   - Controls who can access AWS resources and what they can do.
   - Uses policies to grant or restrict permissions.
   - Supports multi-factor authentication (MFA) for extra security.
   - Free service; integrated with all AWS services. 

3. AWS CloudWatch
   - Manages users, groups, and roles in AWS.
   - Controls who can access AWS resources and what they can do.
   - Uses policies to grant or restrict permissions.
   - Supports multi-factor authentication (MFA) for extra security.
   - Free service; integrated with all AWS services.

4. AWS LAMDA
   - Serverless compute service to run code without managing servers.
   - Executes code automatically in response to events.
   - Supports multiple programming languages (Python, Java, Node.js, etc.).
   - Scales automatically based on the number of requests.
   - Pay only for execution time, not for idle resources.

5. AWS EKS
   - Elastic Kubernetes Service
   - Managed service to run Kubernetes clusters on AWS.
   - Automatically manages Kubernetes control plane.
   - Integrates with EC2, IAM, VPC, and Load Balancers.
   - Provides high availability and security.
   - Used for deploying and scaling containerized applications.
